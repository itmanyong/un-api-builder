# 配置参考

本文档提供 un-api-builder 所有配置项的完整说明。

## 配置结构

```typescript
interface Config {
  /** 文档配置数组 */
  docs?: DocConfig[];
  /** 全局前缀 */
  prefix?: string;
  /** 全局输出路径 */
  output?: string;
}

interface DocConfig {
  /** 文档名称（必需） */
  name: string;
  /** OpenAPI 文档 URL（必需） */
  url: string;
  /** 是否启用 */
  enable?: boolean;
  /** 输出路径 */
  output?: string;
  /** 文档前缀 */
  prefix?: string;
  /** 输出文件类型 */
  outputType?: "ts" | "js";
  /** 代码模式 */
  codeMode?: "proxy" | "memory";
  /** 导出模式 */
  exportMode?: "api" | "module" | "doc" | "default";
  /** 请求器 */
  requestor?: string;
  /** 文件头部注释 */
  fileHeader?: string[] | ((filename: string, type: string, ...args: any[]) => string[]);
  /** 文件尾部注释 */
  fileFooter?: string[] | ((filename: string, type: string, ...args: any[]) => string[]);
  /** 文档解析器 */
  docParser?: (doc: any) => OpenAPIObject;
  /** API 解析器 */
  apiParser?: (ctx: ApiParamsCtx) => DocApiConfig;
  /** API 响应状态码 */
  apiResponseCode?: number | string | ((ctx: ApiParamsCtx) => number | string);
  /** API 响应类型 */
  apiResponseType?: string | ((ctx: ApiParamsCtx) => string);
  /** API 请求类型 */
  apiRequestType?: string | ((ctx: ApiParamsCtx) => string);
  /** 是否启用多请求体类型 */
  enableMultiContentType?: boolean;
  /** 类型生成配置 */
  dts?: DocDtsConfig;
  /** 缓存配置 */
  cache?: DocCacheConfig;
}

interface DocDtsConfig {
  /** 是否启用 */
  enable?: boolean;
  /** 类型文件输出路径 */
  path?: string;
  /** 接口参数泛型 */
  apiOptions?: string;
  /** 接口结果泛型 */
  apiResult?: string;
  /** 文件头部注释 */
  fileHeader?: string[] | ((filename: string, type: string, ...args: any[]) => string[]);
  /** 文件尾部注释 */
  fileFooter?: string[] | ((filename: string, type: string, ...args: any[]) => string[]);
  /** 类型转换配置 */
  typeConvert?: Partial<Record<keyof SchemaObject, (value: SchemaObject | ReferenceObject) => string>>;
  /** 路径别名 */
  pathAlias?: string;
}

interface DocCacheConfig {
  /** 是否启用 */
  enable?: boolean;
  /** 缓存路径 */
  path?: string;
  /** 缓存过期时间（毫秒） */
  expireTime?: number;
}
```

## 配置示例

### 最小配置

```typescript
import { defineConfig } from "un-api-builder";

export default defineConfig({
  docs: [
    {
      name: "my-api",
      url: "https://api.example.com/openapi.json",
    },
  ],
});
```

### 完整配置

```typescript
import { defineConfig } from "un-api-builder";

export default defineConfig({
  docs: [
    {
      name: "my-api",
      url: "https://api.example.com/openapi.json",
      output: "src/apis/my-api",
      prefix: "api",
      outputType: "ts",
      codeMode: "proxy",
      exportMode: "module",
      requestor: "({url,...ops})=>globalThis.fetch(url, ops)",
      apiResponseCode: 200,
      apiResponseType: "application/json",
      apiRequestType: "application/json",
      enableMultiContentType: false,
      fileHeader: ["// Generated by un-api-builder"],
      fileFooter: [],
      dts: {
        enable: true,
        path: "types",
        apiOptions: "RequestInit",
        apiResult: "Response",
        typeConvert: {},
        pathAlias: "@my-api",
      },
      cache: {
        enable: true,
        path: ".un-api-builder",
        expireTime: 600000,
      },
    },
  ],
});
```

## 配置项详解

### 基础配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `name` | `string` | 必需 | 文档名称，用于生成文件名和缓存键 |
| `url` | `string` | 必需 | OpenAPI 文档的 URL 地址 |
| `enable` | `boolean` | `true` | 是否启用此文档配置 |
| `output` | `string` | `src/apis/${name}` | API 代码输出路径 |
| `prefix` | `string` | `api` | API 函数和类型的前缀 |
| `outputType` | `"ts" \| "js"` | `ts` | 输出文件类型 |

### 代码生成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `codeMode` | `"proxy" \| "memory"` | `proxy` | API 调用模式 |
| `exportMode` | `"api" \| "module" \| "doc" \| "default"` | `module` | 导出模式 |
| `requestor` | `string` | `({url,...ops})=>globalThis.fetch(url, ops)` | 请求器函数 |

#### codeMode 代码模式

- **`proxy`（默认）** - 代理模式，使用 Proxy 动态生成 API 调用代码，支持 Vue 响应式跳过标记
- **`memory`** - 内存模式，将 API 定义缓存到内存中，适用于测试场景

#### exportMode 导出模式

- **`api`** - 每个 API 单独导出一个函数
  ```typescript
  export const apiUserGetById = createApiProxy(apis.user.getById);
  export const apiUserCreate = createApiProxy(apis.user.create);
  ```

- **`module`（默认）** - 按模块导出对象
  ```typescript
  export const apiUser = createModuleProxy(apis.user);
  export const apiOrder = createModuleProxy(apis.order);
  ```

- **`doc`** - 具名导出整个文档对象
  ```typescript
  export const apiMyApi = createDocProxy(apis);
  ```

- **`default`** - 默认导出整个文档对象
  ```typescript
  export default createDocProxy(apis);
  ```

### API 配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `apiResponseCode` | `number \| string \| function` | `200` | API 响应成功状态码 |
| `apiResponseType` | `string \| function` | `*/*` | API 响应 Content-Type |
| `apiRequestType` | `string \| function` | `*/*` | API 请求 Content-Type |
| `enableMultiContentType` | `boolean` | `false` | 是否启用多请求体类型支持 |
| `apiParser` | `function` | `null` | API 配置解析器 |

### 类型生成配置 (dts)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dts.enable` | `boolean` | `true` | 是否生成类型文件 |
| `dts.path` | `string` | `types` | 类型文件输出路径 |
| `dts.apiOptions` | `string` | `RequestInit` | 接口参数泛型类型名 |
| `dts.apiResult` | `string` | `Response` | 接口结果泛型类型名 |
| `dts.pathAlias` | `string` | `@${name}` | 路径别名，用于激活全局 TS 类型匹配 |
| `dts.typeConvert` | `object` | `{}` | 自定义类型转换函数 |

#### dts.apiOptions 和 dts.apiResult

这两个配置用于定义 API 函数的泛型参数：

```typescript
// dts.apiOptions = "RequestInit"
// dts.apiResult = "Response"
export const apiUserGetById = (
  options: RequestInit,  // 来自 apiOptions
  params: { id: number }
): Promise<Response> => { ... };  // 来自 apiResult
```

### 缓存配置 (cache)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `cache.enable` | `boolean` | `true` | 是否启用缓存 |
| `cache.path` | `string` | `.un-api-builder` | 缓存文件路径 |
| `cache.expireTime` | `number` | `600000` | 缓存过期时间（毫秒），默认 10 分钟 |

### 解析器配置

#### `docParser`

文档解析器，在获取 OpenAPI 文档后、生成代码前对文档进行处理。支持将其他协议格式转换为 OpenAPI v3+ 规范。

```typescript
docParser: (doc) => {
  // 转换 Swagger 2.0 到 OpenAPI 3.0
  return convertToOpenApi3(doc);
}
```

#### `apiParser`

API 解析器，对每个 API 配置进行处理，返回的配置会合并到 `ctx.config` 中。

```typescript
apiParser: (ctx) => {
  // 添加自定义请求头
  return {
    options: {
      headers: {
        Authorization: `Bearer ${getToken()}`,
      },
    },
  };
}
```

## 配置文件位置

按优先级从高到低：

1. `configPath` 选项指定的路径
2. `un-api-builder.config.ts`
3. `un-api-builder.config.mts`
4. `un-api-builder.config.cts`
5. `un-api-builder.config.js`
6. `un-api-builder.config.mjs`
7. `un-api-builder.config.cjs`
8. `un-api-builder.config.json`
9. `package.json` 中的 `un-api-builder` 字段
